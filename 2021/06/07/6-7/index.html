SQL

定义了操作所有关系型数据库的规则

SQ通用语法

1.以单行或多行书写，以分号结尾。

2.用空格或者缩进进行增强语句的可读性。

3.不区分大小写，但是关键字建议使用大写。

4.三种注释

​ 单行注释：–或者#

​ 多行注释：/* */

5.SQL分类

（1）DDL数据定义语言

​ 关键字：create,drop,alter

（2）DML数据操作语言

​ 关键字：insert,delete,update

（3）DQL数据查询语言

​ 关键字：select,where

（4）DCL数据控制语言(授权)

​ 关键字：GRANT,REVOKE

DDL

1.操作数据库：CRUD

​ 1.C（Create）：创建

​ 创建数据库

​ create database 数据库名称；

​ 创建数据库，判断不存在，在创建

​ create database if not exists 数据库名称；

​ 创建数据库，并指定字符集

​ create database 数据库名称 character set 字符集名；

​ 创建db4数据库。判断是否存在，并指定字符集为gbk;

​ creater database if not exists db4 character set gbk;

​ 2.R（Retrirve）：查询

​ 查询所有数据库的名称：

​ show databases；

​ 查看某个数据库的字符集：查看某个数据库的创建语句

​ show create database；

​ 3.U（Update）：修改

​ 修改数据库的字符集

​ alter database 数据库名称 character set 字符集名称；

​ 4.D（Delete）：删除

​ 删除数据库

​ drop database 数据库名称；

​ 判断数据库是否存在，存在再删除

​ drop database if exists 数据库名称 ；

​ 5.使用数据库

​ 查询当前正在使用的数据库名称

​ select database（）；

​ use 数据库名称；（没有database！！）

操作表

1.C（Create）：创建

​ 1.语法

​ create table 表名（

​ 列名1 数据类型1，

​ 列名2 数据类型2，

​ ……

​ 列名n 数据类型n

​ ）；

​ 注意：最后一列不需要加逗号

​ 数据类型：

​ 1.int：整数类型

​ age int，

​ 2.double：小数类型

​ score double（5，2）（最大值999.99）

​ 3.date：日期：只包含了年月日 yyyy-MM-dd

​ 4.datetime：日期，包含了年月日时分秒 yyyy-MM-dd HH：mm：ss

​ 5.timestamp：时间错类型 包含年月日时分秒 yyyy-MM-dd HH：mm：ss

​ 如果是timestamp类型，如果将来不给这个字段赋值，或赋值为null，则默认使用当前的系统时间来自动赋值。

​ 6.varchar：字符串

​ name varchar（20）；表示姓名最大20字符

​ 创建表：

​ create table student（

​ id int，

​ name carchar(32),

​ age int,

​ score double(4,1),

​ birthday date,

​ insert_time timestamp

​ ）;

​ 复制创建表：

​ create table 新的表名 like 需要复制的表名；

2.R（Retrieve）：查询

​ 查询某个数据库中所有的表名称

​ show tables；

​ 查询表结构

​ desc 表名；

3.U(Update)：

​ 1.修改表名

​ alter table 表名 rename to 新的表名；

​ 2.修改表的字符集

​ alter table 表明character set 字符集名称；

​ 3.修改表的一列（添加）

​ alter table 表名 add 列名 数据类型；

​ 4.修改列名称 类型

​ （1）alter table 表名 changer 列名 修改之后的列名 修改之后的类型

​ （2）alter table 表名 modify 列名 修改之后的类型

​ 5.删除表的一列

​ alter table 表名 drop 列名；

4.D（Delete）：删除

​ drop table 表名；

​ drop table if exists 表名；
