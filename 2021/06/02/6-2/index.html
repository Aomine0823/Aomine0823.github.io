继承中成员方法的访问特点

在父子类的继承关系当中，创建子类对象，访问成员方法的规则

1.创建的对象是谁，就优先用谁，如果没有则向上找

注意事项：

无论是成员方法还是成员变量，如果没有都是向上找父类，绝对不会向下找子类

继承中方法的重写覆盖

重写（override）：方法名称一样，参数列表也一样。（覆盖、覆写）

重载（overload）：方法名称一样，参数列表一样

方法覆盖重写的特点：创建的是什么对象，就调用什么对象重写的

注意事项：

1.必须保证父类和子类的方法名称一样

注解@override，卸载方法前面，用来检测是不是有效的正确覆盖重写。也可以不写，只是用来检验。

2.子类方法的返回值必须小于等于父类方法的返回值范围。

java.long.Object类是所有类的公共最高父类(祖宗类)，java.long.String就是Object的子类。

举例：

父类：public Object method（）

子类：public String method（）

这是可以的，因为String是Object的子类

但是如果子类是void不行，因为父类有返回值，但是子类没有返回值？

3.子类的权限必须大于等于父类方法的权限修饰符

public > protect > (default) > private

default是什么都不写，写了反倒报错？

继承中构造方法的访问特点

1.子类构造方法当中有一个默认隐含的super（）调用，什么都没写，则默认先调用的父类构造，后执行的子类构造。

2.可以通过super关键字来子类构造调用父类重载构造

3.super的父类构造调用，必须是子类构造方法的第一个语句。不能一个子类构造调用多个super

子类必须调用父类构造方法，不写则赠送super（），写了则用写的指定的super调用。super只能有一个还必须要第一个。

super关键字的三种用法

1.在子类的成员方法中，访问父类的成员变量

2.在子类的成员方法中，访问父类的成员方法

3.在子类的构造方法中，访问父类的构造方法

this关键字的三种方法

1.在本类的成员方法中，访问本类的成员变量

2.在本类的成员方法中，访问本类的另一个成员方法（主要是起到强调作用，来子本类而不是父类）

3.在本类的成员方法中，访问本类的另一个构造方法

比如：本来的有参构造调用本类的无参构造。

注意：

1.this（）调用必须是构造方法的第一个语句。唯一一个。

2.super（）和this（）两种构造调用，不同时使用，因为他俩都是唯一的。

3.不能循环调用

Java集成的三个特点

1.Java是单继承的。

一个类的直接父类只能有唯一一个

2.Java语言可以多级继承

抽象类的概念

如果父类当中的方法不确定如何进行{}方法实现，那么这就应该是一个抽象方法

抽象方法和抽象类的格式

1.格式：public abstract void eat();

2.抽象方法所在类，必须是抽象类才行。

3.抽象类里面的方法不一定非得都是抽象方法。

4.不能直接创建new抽象类对象

5.必须用一个子类来继承抽象父类

6.子类必须覆盖重写抽象父类当中所有的抽象方法

7.创建子类对象进行使用

抽象方法和抽象类的注意事项

抽象类的子类，必须重写抽象父类中所有的抽象方法，否则，编译无法通过而报错。除非该子类也是抽象类。

but这时候就要抽象子类的子类完成未完成的抽象方法的具体化。

举例：

abstract Animal中有abstract sleep，abstact eat

在abstract Animal的子类abstract Dog中有eat

在abstract Dog的子类hashiqi中有sleep

接口的定义基本格式

接口就是一种，公共的规范标准

接口中可以包含的内容有：

如果是Java 7

1.常量

2.抽象方法

如果是Java 8 还可以额外包括

3.默认方法

4.静态方法

如果是Java9 还可以包括

5.私有方法

注意事项：

1.接口当中的抽象方法，修饰符必须是两个固定的关键字：public abstract

2.这两个关键字修饰符，可以选择性的忽略。

3.方法中的三要素，可以随意定义

接口的抽象方法使用

1.接口的格式：

public class 实现类名称 implements 接口名称{

​ // …

}

2.接口的实现类必须覆盖重写（实现）接口中所有的抽象方法

实现：去掉abstract关键字，加上方法体大括号

3.创建实现类的对象，进行使用

注意事项：

如果实现类并没有覆盖重写接口中的所有的抽象方法，那么这个实现类自己就必须是抽象类。

Java中接口的默认方法定义与使用

格式：

public default 返回值类型 方法名称（参数列表{

方法体

}

接口当中的默认方法，可以解决接口升级的问题。

（当接口中需要添加新的方法，如果用的是抽象类，则在运用接口的类中都需要添加新的方法就很麻烦。）

默认方法可以被继承类继承下去

1.接口的默认方法，可以通过接口实现类对象，直接调用

2.接口的默认方法，也可以被接口实现类进行覆盖重写

接口的静态方法定义和使用

public static 返回值类型 方法名称（参数列表）{

​ 方法体

}

注意：

不能通过接口实现类的对象来调用接口中的静态方法。因为一个类可以实现多个接口，多个类中的静态方法有可能会产生冲突。

正确用法：

接口名称.静态方法名(参数);

接口的私有方法定义

需要抽取一个公有方法，用来解决两个默认方法之间重复代码的问题。

1.普通私有方法：解决多个默认方法之间重复代码问题

格式：private 返回值类型 方法名称(参数列表){

方法体

}

2.静态私有方法，解决多个静态方法之间的重复代码问题

格式：private static 返回值类型 方法名称(参数列表){

方法体

}

接口的常量定义和使用

必须要用public static final 三个关键字进行修饰

从效果上看，这就是接口的常量。

一旦赋值，就不可以修改。

一旦使用final关键字进行修饰，就不可改变

1.接口中的常量，可以省略public static final, 注意，不写也照样是这样

2.接口当中的常量，必须进行赋值，不能不赋值

3.接口中常量的名称，使用完全大写的字母，用下划线进行分割。

使用方法：

接口名称.常量名称

Java接口内容小结

注意：

1.接口不能有静态方法块

2.接口不能有构造方法

3.一个类的直接父类是唯一的，但是一个类可以同时实现多个接口。

4.如果实现类所实现的多个接口方法，只需要重写覆盖一次

5.如果实现类没有覆盖重写所有接口当中的所有方法，实现类必须是抽象类。

6.如果实现类所实现的多个接口中，存在重复的默认方法，实现类一定要对冲突的默认方法进行覆盖重写。

7.继承优先于接口实现，如果继承和接口的方法重名，并且没有重写方法，优先调用父类的方法。

接口中的多继承

1.类与接口之间是多实现的。一个类可以实现多个接口

2.接口与接口之间是多继承的。

注意事项

1.多个父接口当中的抽象方法如果重复没有问题。

2.多个父接口当中的默认方法如果重复，那么子接口必须进行默认方法的覆盖重写。

多态

多态是指一个对象拥有多种形态。

多态的格式与使用

格式：

父类名称 对象名 = new 子类名称();

或者

接口名称 对象名 = new 实现类名称();

优先运行的子类方法，因为new出来的是子类

多态中成员变量的使用特点

1.直接通过对象名称访问成员变量，看等号左边是谁，优先用水，没有则想上找。

2.间接通过成员方法访问，该方法属于谁，优先用谁，没有则向上找。

3.编译看左，运行也看左

多态中成员方法的使用特点

1.看new的是谁，就优先用谁，没有就向上找

2.编译看左边，运行看右边

即是否有这个方法看左边，运行结果看右边

子类有重写才能用子类，不然全看父类
